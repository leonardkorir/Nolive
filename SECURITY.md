# Security Policy

## Supported Scope

当前安全响应范围以仓库中已提交的主应用、packages 和构建脚本为准。

## Reporting a Vulnerability

如果你发现了安全问题，请不要直接在公开 issue 中披露完整细节。

建议做法：

1. 先最小化描述问题，只说明存在风险，不公开利用细节。
2. 如果仓库启用了 GitHub Private Vulnerability Reporting，请优先使用。
3. 如果没有私密渠道，请先提交最小公开 issue，请维护者提供私下沟通方式，再继续披露细节。

## Do Not Publish Publicly

不要公开以下内容：

- 账号、Cookie、Token、密钥
- 可直接复现攻击的完整 payload
- 任何用户隐私数据
